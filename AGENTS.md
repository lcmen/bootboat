# Agents

## Project Overview

Bootboat is a semantic CSS component library built on Tailwind CSS v4. It provides Bootstrap-like class names (`.btn-primary`, `.form-input`, `.card`) using Tailwind's `@utility` directive.

Visual design based on [Sailboat UI](https://sailboatui.com/) by Jeremie Carlson.

## Key Files

- `bootboat.css` — The library. Single source of truth for all component classes and theme tokens. Uses `@utility` with `:where()` for zero specificity.
- `website/` — Eleventy (11ty) source for the documentation website.
  - `index.html` — Overview page with component grid.
  - `*.html` — Individual component pages (buttons.html, forms.html, etc.).
  - `_includes/layout.html` — Base Eleventy layout with sidebar navigation.
  - `assets/index.css` — Tailwind v4 input file. Imports tailwindcss, @tailwindcss/forms plugin, and bootboat.css. Also contains website-only helper classes.
- `.eleventy.js` — Eleventy configuration. Uses `pathPrefix: "/bootboat/"` in production (when `CI` env var is set) for GitHub Pages subdirectory deployment, otherwise "/" for local development.
- `package.json` — Build scripts and dependencies.
- `dist/` — Build output (generated by Eleventy + Tailwind).

## Architecture

- All component styles live in `bootboat.css` using `@utility name { :where(&) { @apply ...; } }` pattern.
- Theme colors (primary, secondary) and font are defined in `@theme {}` block in `bootboat.css`.
- The website is built with Eleventy (11ty) and uses Alpine.js for interactive components (dropdown, modal, tabs).
- Interactive state is handled via Alpine.js directives (`x-data`, `x-show`, `@click`, etc.).
- The website pages do NOT duplicate bootboat styles — they import them via the build.

## Build System

```
npm run build       # Full production build (clean + CSS + 11ty)
npm run build:css   # Compile CSS only (Tailwind CLI)
npm run build:11ty  # Build static site only (Eleventy)
npm run dev         # Development mode (watch CSS + 11ty serve)
npm run watch:css   # Watch CSS changes only
npm run watch:11ty  # Serve with 11ty dev server + watch
npm run clean       # Remove dist/ directory
```

**Development URL:** Access the site at `http://localhost:8082/` during local development.

## Conventions

- **Naming:** `.component` for base, `.component-variant` for variants, `.component-sm`/`.component-lg` for sizes.
- **Specificity:** All utilities use `:where()` wrapper for zero specificity so Tailwind utilities can override them.
- **Variants:** Use `@variant` for pseudo-classes (hover, focus, disabled).
- **Website-only styles:** Put demo/helpers in `website/assets/index.css`, library styles in `bootboat.css`.
- **No inline styles:** Do not add styles to HTML files inline.

## Theme Tokens

Defaults defined in `bootboat.css` `@theme` block:

- `--color-primary-{50..900}` — Default: blue scale
- `--color-secondary-{50..900}` — Default: slate scale
- `--font-sans` — Default: Inter

Override in your own `@theme` block after importing bootboat.css.

## Dependencies

- `tailwindcss` ^4 — Core framework
- `@tailwindcss/forms` ^0.5 — Form element base styling (use `strategy: base`)
- `@tailwindcss/cli` ^4 — Tailwind CLI for builds
- `@11ty/eleventy` ^3.1.5 — Static site generator for website pages
- `concurrently` ^9 — Run watch tasks in parallel

## Deployment

GitHub Pages via GitHub Actions (`.github/workflows/pages.yml`):
- Triggers on push to `main` or manual dispatch
- Builds with `npm run build`
- Deploys the `dist/` directory
- Uses Node.js 22
